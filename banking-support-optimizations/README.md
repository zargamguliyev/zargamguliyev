# Оптимизация процесса клиентской поддержки банка «Сберегай»

## Обзор проекта

Данный проект посвящён анализу и оптимизации процесса клиентской поддержки регионального банка.

Банк провёл опрос удовлетворённости клиентов, который показал, что средняя оценка качества службы поддержки составила **6,3 из 10**.  
Бизнес-цель — повысить этот показатель до **8 из 10** за счёт улучшения процесса поддержки и внедрения автоматизированного сбора обратной связи.

Проект включает анализ текущего процесса, выявление узких мест и проектирование улучшенного процесса поддержки.

---

# Бизнес-проблема

Интервью с клиентами и внутренний анализ выявили ряд проблем:

- 80% обращений в поддержку поступают по телефону
- клиенты предпочитают письменное общение, но вынуждены звонить
- среднее время ответа на электронное письмо достигает **2 дней**
- операторы тратят **80% рабочего времени** на обработку звонков
- **60% обращений носят повторяющийся характер**
- отсутствует автоматизированный **сбор обратной связи** после завершения обращения

Эти проблемы приводят к:

- перегрузке операторов
- низкой скорости ответов
- снижению удовлетворённости клиентов

---

# Решение

Предлагаемое решение предусматривает внедрение **чат-поддержки внутри мобильного банковского приложения**.

Ключевые элементы решения:

- Чат-поддержка внутри мобильного приложения  
- Чат-бот для обработки типовых обращений  
- Эскалация к оператору поддержки при необходимости  
- Автоматизированный сбор обратной связи после закрытия обращения

Данный подход снижает нагрузку на операторов и повышает скорость обработки запросов.

# Моделирование бизнес-процессов

## Процесс AS-IS

BPMN-модель AS-IS отражает текущий процесс обработки обращений клиентов через:

- телефонную поддержку
- электронную почту

Диаграмма наглядно демонстрирует основные неэффективности существующего процесса.

## Процесс TO-BE

Модель TO-BE предусматривает:

- чат-поддержку
- автоматизацию с помощью чат-бота
- сбор обратной связи
- ускоренное решение обращений

---

# User Story Map

User Story Mapping применялся для определения:

- основного пути пользователя
- ключевых возможностей системы
- функциональных требований к новому решению

Основные действия пользователя:

- обращение в поддержку через чат
- получение автоматических ответов
- эскалация к оператору
- оставление отзыва после закрытия обращения

---

# Lo-Fi прототип

Прототип низкой точности разработан для демонстрации будущего интерфейса системы поддержки.

### Основные экраны

**Экран авторизации**

- логотип банка
- поля для ввода логина и пароля
- кнопка входа
- быстрый доступ к поддержке

**Экран выбора способа связи**

Пользователь может выбрать удобный канал коммуникации:

- чат-бот
- телефонный звонок
- электронная почта

**Интерфейс чат-бота**

Пользователь может быстро выбрать типовое обращение:

- выписка по счёту
- информация об отделениях
- восстановление пароля
- отмена подписки
- акции и предложения
- другой вопрос
- связаться с оператором

**Общение с оператором**

Если чат-бот не может решить вопрос, пользователь может обратиться к оператору поддержки.

**Сбор обратной связи**

После решения вопроса пользователь может оценить качество обслуживания.
